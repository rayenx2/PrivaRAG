How It Works
Four steps from raw document to grounded, cited answer.
1

Ingest

Upload any document via the React UI. The FastAPI backend stores it and kicks off an async processing task using PyMuPDF + Tesseract OCR.

2

Embed

The text is chunked (1 000 chars, 100 overlap) by LangChain's RecursiveCharacterTextSplitter. Each chunk is encoded with BAAI/bge-m3 (1 024-d, GPU-accelerated) and stored in Qdrant.

3

Retrieve

Your query is embedded with the same model. Qdrant performs an approximate nearest-neighbour search (cosine similarity, threshold 0.30) and returns the top-15 most relevant chunks.

4

Generate

Retrieved chunks are injected into a structured prompt template. Ollama runs Qwen3-14B (Q4_K_M) fully locally. The answer is returned with source filenames and similarity scores.

Query the RAG System
Python one-liner using the REST API.
Python — query_example.py
import requests # 1. Authenticate token = requests.post( "http://localhost:8000/api/auth/login", json={"username": "admin", "password": "<from-docker-logs>"} ).json()["access_token"] headers = {"Authorization": f"Bearer {token}"} # 2. Upload a document with open("report.pdf", "rb") as f: requests.post( "http://localhost:8000/api/documents/upload", files={"file": f}, headers=headers ) # 3. Run a RAG query response = requests.post( "http://localhost:8000/api/query", json={ "query": "What were the key findings in Q2 2025?", "top_k": 5, "temperature": 0.0 }, headers=headers ).json() print(response["answer"]) # Cited answer from local LLM print(response["sources"]) # [{"filename": ..., "similarity_score": ...}] print(response["processing_time"]) # e.g. 1.84 seconds # 4. Evaluate RAG quality (new in this portfolio) from scripts.rag_evaluator import RAGEvaluator evaluator = RAGEvaluator() result = evaluator.evaluate( query="What were the key findings in Q2 2025?", answer=response["answer"], contexts=[s["text"] for s in response["sources"]] ) print(result.summary()) # Grade + faithfulness + relevance scores
Local RAG vs API-Based RAG
Why running RAG locally is the right choice for enterprise knowledge bases.
Criterion PrivaRAG (This Project) OpenAI RAG / Azure AI Search
Data Privacy Data never leaves server Data sent to cloud
Cost per Query $0 — electricity only $0.002–$0.06 per query
Retrieval Latency (p50) < 2s on RTX 3050 1–4s (network dependent)
Model Customization Any Ollama model, swap anytime Limited to vendor models
Offline Operation Fully offline after setup Requires internet connection
GDPR / DSGVO Compliance Trivial — no data transfer Requires DPA + legal review
Scale (documents) Up to ~100K chunks on single GPU Unlimited (at cost)
Setup Complexity docker compose up (5 min) Managed service (minutes)
